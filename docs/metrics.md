# 指标设计说明

## 指标设计原则

- 优先选择能体现大数据处理流程的指标。
- 同时兼顾实时分析价值和离线分析价值。
- 指标数量控制在课程展示可讲清楚的范围内。

## 六个核心指标

### 1. 实时事件流监控

- 指标含义：每分钟 GitHub 事件总数和分类型事件数量。
- 数据来源：`Kafka -> Flink`
- 展示价值：最直观体现实时处理能力。

### 2. 热门仓库榜单

- 指标含义：按加权热度分数评估当前窗口的热门仓库。
- 评分示例：
- `WatchEvent * 1.0`
- `ForkEvent * 3.0`
- `IssuesEvent * 2.0`
- `PullRequestEvent * 2.5`
- `PushEvent * 1.5`
- 数据来源：`Kafka -> Flink -> ClickHouse`
- 展示价值：适合做榜单和排名，演示效果好。

### 3. 历史趋势分析

- 指标含义：按天统计总事件量、仓库活跃度和事件分布。
- 数据来源：`Spark`
- 展示价值：体现离线全量分析能力。

### 4. 异常活跃检测

- 指标含义：检测短时间内事件量明显高于基线的仓库。
- 简化方法：当前窗口事件数 / 平均事件数。
- 数据来源：`Flink`
- 展示价值：体现“预测/预警型”分析思路。

### 5. 开发者活跃节律

- 指标含义：按小时和星期维度统计开发活动模式。
- 数据来源：`Spark`
- 展示价值：属于有解释力的行为画像指标，容易体现新意。

### 6. 人类与 Bot 行为对比

- 指标含义：比较 Bot 与人类账号的事件量占比。
- 数据来源：`Flink + Spark`
- 展示价值：体现对数据语义的理解，不只是计数。

## 可选创新指标

### 仓库热度突增预警

- 将异常活跃检测包装为更具产品感的预警功能。
- 适合在演示时作为亮点。

### 长期稳定活跃 vs 短期爆发活跃

- 用离线长期趋势和实时窗口热度做对比。
- 能清楚展示“流批结合”的价值。
