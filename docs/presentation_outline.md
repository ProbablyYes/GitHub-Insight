# 答辩展示提纲

## 1. 开场

- 项目名称：GitHub 开发者行为流批分析系统
- 课程目标：展示大数据技术在真实公开数据上的应用

## 2. 为什么选这个题目

- GitHub 数据公开、规模大、容易形成 GB 级数据集
- 同时适合做实时分析和离线分析
- 结果可视化强，适合演示

## 3. 系统架构

- GH Archive 下载
- Kafka 事件总线
- Flink 实时处理
- Spark 离线分析
- ClickHouse 统一查询
- Superset / Streamlit 展示

## 4. 核心功能展示

- 实时事件流监控
- 热门仓库榜单
- 异常活跃预警
- 日级趋势图
- Bot 与人类账号对比
- 开发者活跃节律

## 5. 创新点

- 历史事件回放模拟实时分析
- 同时展示流处理和批处理结果
- 在课程项目规模下实现可视化分析平台

## 6. 难点与解决方案

- 数据规模控制
- 组件部署复杂度
- 演示环境稳定性
- Flink 正式方案与轻量本地方案并存

## 7. 总结

- 系统满足课程对大数据技术应用的要求
- 能展示完整的大数据处理链路
- 可进一步扩展为更正式的开源生态分析平台
